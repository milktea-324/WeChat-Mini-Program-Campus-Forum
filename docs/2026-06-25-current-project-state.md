# 校园论坛小程序当前稳定版本说明

## 1. 当前项目定位

当前项目是一个基于微信小程序本地缓存实现的校园论坛项目，面向期末课程展示和功能演示。项目不依赖真实后端、不依赖数据库，帖子、评论和我的发布记录都保存在小程序本地缓存中。

当前版本重点是保证校园论坛核心流程可演示、页面功能稳定、数据结构相对清晰，适合作为期末展示前的稳定版本。

## 2. 当前页面结构

| 页面 | 路径 | 主要功能 |
| --- | --- | --- |
| 首页 | `pages/index/index` | 展示校园帖子列表、热帖轮播、搜索、分类筛选和排序入口 |
| 详情页 | `pages/detail/detail` | 展示帖子详情，处理浏览数、点赞、收藏、评论列表和新增评论 |
| 发布页 | `pages/publish/publish` | 发布新帖子，支持标题、分类、正文、联系方式和封面图片 |
| 我的页 | `pages/mine/mine` | 展示我的发布、我的收藏、我的点赞，并支持搜索、分类筛选和排序 |
| 作者详情页 | `pages/user/user` | 根据作者信息展示作者资料、作者统计和 TA 的发布列表 |

## 3. 当前核心功能

当前稳定版本已具备以下核心功能：

* 首页帖子列表：从 `forum-store` 获取帖子数据并渲染帖子列表。
* 热帖轮播：首页根据热度计算结果选取前三条帖子作为轮播内容。
* 搜索：支持按作者、标题、正文进行搜索匹配。
* 分类筛选：支持按校园分类筛选帖子。
* 排序：支持默认热度、最新发布、最早发布、浏览量最高、点赞最多、收藏最多。
* 帖子详情：可进入详情页查看标题、作者、分类、正文、封面和统计数据。
* 浏览数：进入详情页时增加帖子浏览数。
* 点赞：详情页支持点赞和取消点赞，并同步帖子点赞数。
* 收藏：详情页支持收藏和取消收藏，并同步帖子收藏数。
* 评论：详情页评论列表来自独立评论存储，新增评论会写入 `forum_comments`。
* 发布帖子：发布页可新增帖子，并记录到本地帖子列表和我的发布 ID 列表。
* 我的发布：我的页按当前用户发布标识和 `forum_my_posts` 展示帖子。
* 我的收藏：我的页按 `isCollected` 展示已收藏帖子。
* 我的点赞：我的页按 `isLiked` 展示已点赞帖子。
* 作者主页：点击非当前用户作者头像可进入作者主页，查看作者资料、统计和 TA 的发布。

## 4. 当前数据访问结构

| 文件 | 当前职责 |
| --- | --- |
| `utils/forum-store.js` | 帖子数据访问层。负责读取、归一化、保存帖子；提供获取帖子、查找帖子、更新单条帖子、新增帖子、维护我的发布 ID 等能力；主要管理 `forum_posts` 和 `forum_my_posts`。 |
| `utils/comment-store.js` | 独立评论存储层。负责评论对象归一化、从旧帖子评论迁移评论、读取和保存 `forum_comments`、按帖子读取评论、统计有效评论数、创建一级评论、新增评论。 |
| `utils/mock-comments.js` | 模拟评论补齐工具。根据帖子 `commentCount` 为旧数据或初始数据生成模拟评论，并保留 `comments` 兼容字段。 |
| `utils/mock-users.js` | 模拟用户和作者信息工具。根据帖子作者生成 `authorId`、`authorInfo`、作者资料、作者发布列表和作者统计；作者评论统计优先使用 `commentCount`。 |
| `utils/post-filter.js` | 帖子筛选和排序工具。提供时间范围、排序选项、热度计算、搜索匹配、分类筛选和统一排序能力。 |

## 5. 当前本地缓存 key

| key | 存储内容 |
| --- | --- |
| `forum_posts` | 帖子主体数组，包含标题、正文、分类、作者、封面、浏览数、点赞数、收藏数、`commentCount`、兼容评论字段等。 |
| `forum_my_posts` | 当前用户发布过的帖子 ID 数组，用于我的页识别“我的发布”。 |
| `forum_comments` | 独立评论对象数组，详情页评论列表和新增评论已接入该 key。 |

## 6. 当前帖子对象结构

当前帖子主要字段如下：

| 字段 | 说明 |
| --- | --- |
| `postId` | 帖子 ID |
| `title` | 帖子标题 |
| `content` | 帖子正文 |
| `category` | 帖子分类 |
| `author` | 作者昵称 |
| `avatar` | 作者头像 |
| `postImg` | 帖子封面图片路径，空字符串表示无封面 |
| `date` | 发布日期 |
| `view` | 浏览数 |
| `like` | 点赞数 |
| `collect` | 收藏数 |
| `commentCount` | 评论数量，当前列表、热度和作者统计的主要口径 |
| `isLiked` | 当前用户是否已点赞 |
| `isCollected` | 当前用户是否已收藏 |
| `isMine` | 是否为当前用户发布 |
| `comments` | 评论兼容字段，当前仍保留用于旧数据兼容和迁移 |
| `authorId` | 作者 ID，由 `mock-users` 补齐 |
| `authorInfo` | 作者摘要信息，由 `mock-users` 补齐 |

说明：`comments` 当前仍作为兼容字段保留。详情页评论展示已经改为读取 `comment-store`，但帖子对象中仍同步 `comments`，用于旧缓存兼容和结构过渡。

## 7. 当前评论对象结构

当前评论主要字段如下：

| 字段 | 说明 |
| --- | --- |
| `commentId` | 评论 ID |
| `postId` | 所属帖子 ID |
| `authorId` | 评论作者 ID |
| `author` | 评论作者昵称 |
| `avatar` | 评论作者头像 |
| `content` | 评论正文 |
| `createdAt` | 评论创建时间 |
| `date` | 页面展示用日期 |
| `parentCommentId` | 父评论 ID，当前一级评论为 `null` |
| `rootCommentId` | 根评论 ID，当前一级评论为 `null` |
| `level` | 评论层级，当前一级评论为 `1` |
| `likeCount` | 评论点赞数，当前未启用交互 |
| `dislikeCount` | 评论点踩数，当前未启用交互 |
| `isLiked` | 当前用户是否点赞该评论，当前未启用交互 |
| `isDisliked` | 当前用户是否点踩该评论，当前未启用交互 |
| `status` | 评论状态，当前有效评论默认为 `active` |

当前只实现一级评论，尚未实现评论回复、评论点赞、评论点踩。

## 8. 当前评论数量口径

当前评论数量口径已经收敛为优先使用 `post.commentCount`：

* 页面列表和热度排序主要依赖 `post.commentCount`。
* 作者统计优先使用 `post.commentCount`。
* `post.comments.length` 只作为旧数据兼容回退。
* 详情页评论列表来自 `comment-store`，即从 `forum_comments` 中按 `postId` 读取。
* 新增评论时，评论写入 `forum_comments`，同时同步更新帖子对象中的 `commentCount`。

## 9. 当前已完成的结构改造

1. 帖子数据访问层收敛：新增并接入 `forum-store`，减少页面直接读写帖子缓存。
2. 评论对象分离设计：明确独立评论对象字段，为后续回复、点赞、点踩预留结构。
3. 评论存储层第一阶段：新增 `comment-store`，支持独立评论缓存、迁移、读取和计数。
4. 详情页评论接入独立评论存储：评论列表改从 `comment-store` 获取，新增评论写入 `forum_comments`。
5. 评论数量统计口径收敛：页面、热度排序和作者统计优先使用 `commentCount`，`comments.length` 仅作兼容回退。

## 10. 当前未做的功能

当前稳定版本明确未做以下功能：

* 未接入后端。
* 未接入数据库。
* 未做真实登录。
* 未做评论回复。
* 未做评论点赞 / 点踩。
* 未拆分帖子点赞收藏关系表。
* 未拆分独立用户表。
* 未做标签系统。
* 未做内容审核。

## 11. 期末展示建议流程

建议展示顺序如下：

1. 打开首页，说明这是本地缓存版校园论坛。
2. 展示顶部热帖轮播，说明热度来自浏览、点赞、收藏、评论数量综合计算。
3. 使用搜索框搜索帖子，展示搜索匹配结果。
4. 使用分类筛选和排序，展示列表筛选能力。
5. 点击帖子进入详情页，展示帖子正文、封面和统计信息。
6. 在详情页进行点赞、收藏、评论，展示交互结果。
7. 进入发布页，发布一条新帖子。
8. 回到首页查看新帖子出现在列表中。
9. 进入我的页，查看我的发布、我的收藏、我的点赞。
10. 点击其他作者头像进入作者主页，查看作者信息和 TA 的发布。

## 12. 后续开发建议

从期末展示角度，建议后续优先级如下：

1. 补图标和资源，保证页面视觉完整，避免演示时出现空图或资源缺失。
2. 检查首次运行和清空缓存后的表现，确认初始帖子、模拟评论、作者信息都能正常生成。
3. 整理技术文档，重点说明本地缓存、数据访问层、评论对象分离和页面流程。
4. 准备演示数据，确保搜索、分类、排序、评论、作者主页都有足够样例。
5. 最后再考虑评论回复、标签、后端接入等扩展功能，避免在期末展示前继续扩大改造范围。
